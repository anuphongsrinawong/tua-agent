## Phase 2: AI Intelligence Boost 🧠

Inspired by [Gemini brainstorming session](https://share.gemini.google/NepzB6CQ25sZ) on specializing Tau for Rust.

### P0 — Must Have (high impact, low/medium effort)

| # | Feature | Effort | Impact | Description |
|---|---------|:---:|:---:|---|
| 13 | **Self-Correction Loop** | Medium | 🔥🔥🔥 | After agent writes/edits a `.rs` file → auto-run `cargo check` → feed compiler errors back into agent context for self-fix before reporting completion |
| 14 | **Chain-of-Thought** | Low | 🔥🔥🔥 | Add `<thinking>...</thinking>` directive to Rust system prompt — force AI to analyze ownership/lifetimes/architecture before writing code. Reduces hallucination |
| 15 | **rustc --explain** | Low | 🔥🔥 | Detect `EXXXX` error codes in cargo output → run `rustc --explain EXXXX` → inject official compiler explanation into agent context. Cuts debug loops by 50%+ |
| 16 | **Session Checkpointing** | Medium | 🔥🔥 | Auto `git commit` when `cargo check` passes → `/rollback` to previous checkpoint, `/undo` to revert last action. Uses git under the hood |

### P1 — Should Have (medium impact)

| # | Feature | Effort | Impact | Description |
|---|---------|:---:|:---:|---|
| 17 | **Token Budgeting** | Low | 🔥🔥 | Real-time token/cost display in footer + warning when approaching context limit. Shows input/output/cache tokens separately |
| 18 | **Prompt Caching** | Medium | 🔥🔥 | Reorder messages to match Anthropic/OpenAI cache breakpoints. System prompt + tools at top, static project context next, dynamic conversation last. 2-4x faster, cheaper |
| 19 | **Multi-Agent Review** | Medium | 🔥🔥 | Spawn a reviewer sub-agent (cheap model) to audit code before displaying to user. Catches bugs, security issues, style violations. All in background |

### P2 — Nice to Have (v0.0.3+)

| # | Feature | Effort | Impact | Description |
|---|---------|:---:|:---:|---|
| 20 | cargo metadata graph | Medium | 🔥 | `cargo metadata` → dependency graph → only send relevant crate context to AI. Prevents context bloat on large workspaces |
| 21 | cargo expand tool | Low | 🔥 | New tool: `cargo_expand(module)` → shows desugared Rust after macro expansion. Critical for debugging derive macros and proc macros |
| 22 | Memory Hierarchy | High | 🔥🔥 | 3-tier memory: Short-term (last 5 turns), Working (task summary), Long-term (SQLite: bug patterns, coding style). Summarization + keyword retrieval |

### P3 — Future Vision (v0.1.0+)

| # | Feature | Effort | Impact | Description |
|---|---------|:---:|:---:|---|
| 23 | LSP rust-analyzer | High | 🔥🔥🔥 | Talk to rust-analyzer via LSP protocol. Go-to-definition, find references, type hints. Makes agent understand large codebases |
| 24 | Tree-sitter Context | High | 🔥🔥 | Use tree-sitter to extract code skeleton (function signatures, struct/enum defs, imports). Send 10% of tokens for 100% structural awareness |
| 25 | Hierarchical Agents | High | 🔥🔥🔥 | Manager agent → routes to Rust Expert / DevOps / Reviewer sub-agents. Each sub-agent has minimal tools and prompt = faster, cheaper |
| 26 | State Machine Loop | High | 🔥🔥 | FSM agent loop: Planning → Executing → Verifying. No tool overlap between states. Prevents infinite loops |
